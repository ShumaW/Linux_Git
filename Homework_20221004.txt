1. Укажите след. команды:

для полной очистки индекса - git reset

для просмотра разницы между раб.каталогом и посл. коммитом - git diff

для создания новой ветки и одновременного переключения в на нее - git checkout -b <название новой ветки> 

для сравнения веток между собой - git diff <название ветки> <название новой ветки> , git diff ..<название новой ветки> , git ...<название новой ветки>

для удаления локальной ветки - git branch -d <название ветки> (при условии предварительного слияния!!!) и git branch -D (принудительное удаление без предварительного слияния) 

для слияния веток - git merge

для перезаписи посл/коммита - git commit --amend

2. Коротко опишите причину возникновения конфликтов при слиянии и порядок действий для их решения.

Конфликт при  слиянии веток может возникнуть по неосторожности или не внимательности, когда были добавлены изменения в обоих ветках в один и тот же файл в теже строки. После попытки слияния выпадет ошибка. Нужно зайти в файл в любом удобном редакторе кода и вручную внести изменения. Либо удалить один из вариантов кода, либо оставить оба. После этого нужно проиндексировать файл с помощью команды git add и зафиксировать исправления с помощью команды git commit -m 'комментарии'.
